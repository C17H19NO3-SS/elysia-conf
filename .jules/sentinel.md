## 2023-10-27 - [Prototype Pollution via Lexer Parser]
**Vulnerability:** Prototype pollution vector found in `src/lib/parser.ts` where arbitrary keys parsed from the configuration format (like `__proto__`, `constructor`, `prototype`) could be assigned directly to object structures (`config` or `currentBlock`).
**Learning:** Because the parser maps configuration keys to Javascript object properties natively without sanitization, malicious blocks or assignments allow attackers to pollute the global Object prototype context across the Node/Bun environment.
**Prevention:** Explicit validation must be enforced on all dynamic key variables fetched from token streams before they are used as object properties to ensure they are not reserved Javascript metaproperties.
## 2024-05-18 - [Path Traversal in Custom Error Configurations]
**Vulnerability:** A directory traversal vulnerability allowed unauthorized actors or insecure configurations to serve arbitrary local files outside the designated project boundaries via the `errors` directive, due to unvalidated `Bun.file()` usage reading configuration-specified absolute or relative paths.
**Learning:** Using `Bun.file()` without validating the resolved output directory against a safe base directory easily leads to LFI (Local File Inclusion). When accepting custom configurations for files, it's essential to enforce that those paths stay within a permitted directory (such as `process.cwd()` or `public`). Absolute path overrides and `../` characters can bypass naive concatenation.
**Prevention:** Always use `path.resolve` or similar canonicalization logic to build the absolute path, and verify that the final path `.startsWith()` the expected base directory with trailing slash (`baseDir + sep`) to block both `..` injection and absolute paths meant to escape the context. Always wrap file loading inside error boundaries with secure failure states that don't leak stack traces.
